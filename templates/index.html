Welcome to Chatosaurus!

What is this?

This chat application project is my first attempt to learn about how web sockets work. It is based on the following :

  • Flask (also WTForms, Jinja templating etc.): When I decided to do this project I read about Django and Flask, and ended up picking the latter. Django seemed a bit too high-level for me (a beginner), a lot of features come built-in and you don't -need to- understand everything. On the other hand, Flask has a bare-bones approach and you explicitly write down all your route functions yourself, handle exceptions, do redirections, write your decorator functions to handle the repeating patterns. This yields better understanding of the concepts and anyone that knows Python can easily understand everything that is happening and why it is happening. Plus, the official flask documentation is very nice and concise. It took me less than an hour to see that it had everything I need (and a lot more) so I said why not.

  • Socket.IO and Flask-SocketIO: To do the communications between the server and the clients Socket.IO seems the way to go. While going over the Flask Web Development book, I found the author of the book himself has a Python package for Socket.IO integration into Flask applications. Flask-SocketIO documentation is also very nice and concise.

  • python-MySQLdb: I added the mysql connection to learn about how to work with databases in the web application. It felt this plays well with the chat app; you keep an account of registered users and their chat logs.

  • Python Passlib: If I am asking people to register, I thought I might as well store an encrypted password.

  • Bootstrap: When I did the first version of this app, it looked like 1996. Bootstrap takes no time to integrate and looks a lot better.

If there is no link to the source code I must be busy cleaning it up. I will put it right over here.

What is there and what is coming?

The following functionality should be available:

  • Go to the chat room directly with an auto-generated guest user name.

  • Register/login to go to the chat room with your user name. When you register/login the header should change accordingly.

  • Chat room shows the current logged in users in the room on the right hand side.

  • When logged in, go to your profile page to see your recent chat logs. Chat logs of the guest users are also logged if a registered user is with them in the same room, so that a registered user can retrieve all their chat logs with all users. If all the active users in the room are guests, there is no logging.

Tentative to-do list:

  • Make it look better, especially on the mobile.

  • Lobby is the only room so far. Allow the logged in users to select the room to join.

  • Show current room list and the number of users in each room on the chat room page.

  • Show whether a user is typing or not. This sounds interesting because typing is a continuous event. All events I handled so far were all discrete events (join or leave a room, send a message). What do you do with a continuous event? Do you do something like "catch the event, emit, sleep 1 second before checking it again" or is there a better way?

  • Allow the user to search within the chat logs, filter by room and user. Some grouping/organization while displaying these to the user.
